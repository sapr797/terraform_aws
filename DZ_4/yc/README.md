# Домашнее задание к занятию «Организация сети»

## Обязательная часть: Yandex Cloud

### Цель
Создать изолированную сетевую инфраструктуру в Yandex Cloud:
- VPC с публичной и приватной подсетями.
- NAT-инстанс для доступа приватных ресурсов в интернет.
- Виртуальные машины для проверки связности.

### Выполненные шаги

1. **Сеть VPC**  
   - Создана сеть `main-vpc` с CIDR 192.168.0.0/16.
   - Публичная подсеть `public` (192.168.10.0/24) в зоне `ru-central1-a`.
   - Приватная подсеть `private` (192.168.20.0/24) в зоне `ru-central1-a`.

2. **NAT-инстанс**  
   - Развёрнут экземпляр с образом NAT-instance (image_id `fd80mrhj8fl2oe87o4e1`).
   - Присвоен фиксированный IP 192.168.10.254 в публичной подсети.
   - Настроена таблица маршрутов для приватной подсети, направляющая весь исходящий трафик через NAT-инстанс.

3. **Виртуальные машины**  
   - **Публичная ВМ** (`public-vm`) в подсети `public` с публичным IP и доступом в интернет.
   - **Приватная ВМ** (`private-vm`) в подсети `private` без публичного IP, но с доступом в интернет через NAT-инстанс.
   - Обе ВМ созданы на основе Ubuntu 20.04 LTS с использованием актуального образа.

4. **Проверка**  
   - Подключение к публичной ВМ по SSH и выполнение `curl ifconfig.me` подтвердило наличие интернета.
   - С публичной ВМ выполнено подключение к приватной ВМ по внутреннему IP, и также проверен выход в интернет.

### Файлы конфигурации

- `main.tf` – основные ресурсы (VPC, подсети, NAT, ВМ).
- `variables.tf` – объявление переменных (токен, cloud_id, folder_id, путь к SSH-ключу).
- `terraform.tfvars.example` – пример файла с переменными (не загружается в репозиторий).
- `README.md` – данное описание.

Дополнительная часть: AWS (теоретическая)
Из-за санкционных ограничений, препятствующих регистрации в AWS и получению учётных данных, практическое развёртывание инфраструктуры не выполнялось. Однако подготовлены Terraform-конфигурации, полностью соответствующие требованиям задания:

VPC с публичной и приватной подсетями.

Internet Gateway и NAT Gateway.

Security Groups для SSH и ICMP.

Виртуальная машина в публичной подсети и виртуальная машина в приватной подсети с доступом через NAT.

Файлы находятся в папке aws/. Для развёртывания необходимо:

Настроить учётные данные AWS (переменные окружения или ~/.aws/credentials).

Заполнить terraform.tfvars (пример в terraform.tfvars.example).

Выполнить terraform init && terraform apply.

Код позволяет создать инфраструктуру, аналогичную Yandex Cloud, с использованием ресурсов AWS.
