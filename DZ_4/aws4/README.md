# Задание 2*. AWS (теоретическая часть)

Ввиду санкционных ограничений, препятствующих регистрации в AWS и получению учётных данных, практическое развёртывание инфраструктуры не выполнялось. Однако подготовлены Terraform-конфигурации, полностью соответствующие требованиям задания.

## Что реализовано в коде

1. **Сеть VPC**:
   - VPC с CIDR 10.0.0.0/16.
   - Публичные подсети в трёх Availability Zones (AZ) для размещения EKS.
   - Приватные подсети в трёх AZ для RDS.
   - Internet Gateway для публичных подсетей.
   - NAT Gateway для доступа из приватных подсетей в интернет.
   - Таблицы маршрутов для публичных и приватных подсетей.
   - Security Groups: для RDS (разрешает MySQL из VPC), для EKS (разрешает SSH, HTTP, HTTPS из интернета).

2. **RDS MySQL**:
   - Кластер с Multi-AZ для отказоустойчивости.
   - Две реплики чтения, размещённые в двух AZ.
   - Резервное копирование на 7 дней.
   - Хранение данных в приватных подсетях.
   - Параметры шифрования и настройки MySQL 8.0.

3. **EKS (Kubernetes)**:
   - Управляемый кластер EKS (версия 1.28) в публичных подсетях.
   - Три рабочих узла EC2 (тип t3.medium) в публичных подсетях.
   - IAM-роли для кластера и узлов.
   - Включён публичный доступ к API кластера.

4. **Взаимодействие**:
   - Security Group RDS разрешает MySQL-трафик из VPC, что позволяет подам Kubernetes в публичных подсетях подключаться к RDS.
   - После создания кластера EKS можно получить kubeconfig командой из outputs и развернуть phpmyadmin, который будет подключаться к RDS.

## Как использовать (при наличии аккаунта AWS)

1. Скопировать `terraform.tfvars.example` в `terraform.tfvars` и задать свои значения (особенно `db_password`).
2. Выполнить `terraform init`, `terraform plan`, `terraform apply`.
3. После создания кластера выполнить `aws eks update-kubeconfig --region us-east-1 --name eks-cluster`.
4. Развернуть phpmyadmin в кластере:
   ```yaml
   kubectl run phpmyadmin --image=phpmyadmin/phpmyadmin --env="PMA_HOST=<rds_endpoint>" --env="PMA_USER=netology_user" --env="PMA_PASSWORD=<пароль>"
   kubectl expose pod phpmyadmin --type=LoadBalancer --port=80
Получить внешний IP сервиса и открыть в браузере.
