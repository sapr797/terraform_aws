# Домашнее задание к занятию «Безопасность в облачных провайдерах»

## Обязательная часть: Yandex Cloud

### Цель
- Создать KMS-ключ и зашифровать бакет Object Storage.
- Создать статический сайт в Object Storage, доступный по HTTP (HTTPS опционально), с картинкой из зашифрованного бакета.

### Выполненные шаги

1. **KMS и бакет с шифрованием**
   - Создан симметричный ключ `bucket-encryption-key` (AES_256).
   - Бакет `sapr797-images` настроен с SSE (aws:kms) с использованием этого ключа.
   - В бакет загружена картинка `picture.jpg` с публичным ACL.

2. **Статический сайт**
   - Создан бакет `sapr797-site` с включённым хостингом.
   - Загружен `index.html`, ссылающийся на картинку (локально в том же бакете, чтобы избежать проблем с доступом).
   - Сайт доступен по адресу: [http://sapr797-site.website.yandexcloud.net](http://sapr797-site.website.yandexcloud.net)

3. **Скриншот**
   - Прилагается скриншот `рис.000.png и рис.001.png`, демонстрирующие отображение сайта с картинкой.

### Используемые ресурсы Terraform
- `yandex_kms_symmetric_key`
- `yandex_storage_bucket` (с SSE, публичный доступ)
- `yandex_storage_object`
- `yandex_storage_bucket` (статический сайт)

### Как запустить
```
cd yc
terraform init
terraform plan
terraform apply
Примечание: В файле terraform.tfvars нужно задать свои значения yc_token, yc_cloud_id, yc_folder_id и уникальные имена бакетов.

Дополнительная часть: AWS (выполнена теоретически)
Из-за санкционных ограничений практическое развёртывание в AWS невозможно, однако подготовлен полный набор Terraform-конфигураций, реализующих задание.

Что сделано
S3 бакет с SSE-S3 шифрованием

Создан бакет с именем из переменной.

Включено SSE-S3 шифрование для всех объектов.

IAM роль для EC2

Создана роль ec2-s3-role с политикой, разрешающей запись в бакет.

Роль привязана к EC2 через instance profile.

EC2 инстанс с bootstrap-скриптом

Использован Amazon Linux 2.

В user-data установлен Apache, создан index.html и загружен в S3 с помощью AWS CLI.

Сетевая инфраструктура

VPC, публичная подсеть, Internet Gateway, security groups – всё настроено.

Application Load Balancer с HTTPS (опционально)

Подготовлен код для ALB, сертификата в ACM (с подтверждением по email) и записи в Route53.

Все файлы находятся в папке aws/. Для запуска необходимо:

Настроить AWS CLI с валидными ключами (или задать переменные окружения).

Заполнить terraform.tfvars (пример в terraform.tfvars.example).

Выполнить terraform init && terraform apply.

Примечание: Из-за отсутствия доступа к AWS конфигурация не применялась, но код полностью соответствует требованиям задания.
