# Домашнее задание к занятию «Организация сети»

## Обязательная часть: Yandex Cloud

### Цель
Создать изолированную сетевую инфраструктуру в Yandex Cloud:
- VPC с публичной и приватной подсетями.
- NAT-инстанс для доступа приватных ресурсов в интернет.
- Виртуальные машины для проверки связности.

### Предварительные требования
- Установленный [Terraform](https://www.terraform.io/downloads) (версия >= 1.0).
- Аккаунт Yandex Cloud, [зарегистрированный](https://cloud.yandex.ru/) и [авторизованный](https://cloud.yandex.ru/docs/iam/operations/authorized-key).
- Настроенные переменные окружения для провайдера:
  ```
  export YC_TOKEN=$(yc iam create-token)   # или используйте IAM-токен
  export YC_CLOUD_ID="ваш_cloud_id"
  export YC_FOLDER_ID="ваша_folder_id"
  
Дополнительная часть: AWS (не выполнена из-за санкционных ограничений)
В соответствии с заданием, была подготовлена конфигурация Terraform для развёртывания инфраструктуры в AWS. В папке DZ_1/aws/ находятся следующие файлы:

main_aws.tf – конфигурация провайдера AWS, VPC, подсетей, Internet Gateway, NAT Gateway, групп безопасности и виртуальных машин.

variables_aws.tf – объявления переменных для AWS (регион, зона доступности, имя ключевой пары).

Однако из-за санкционных ограничений, введённых в отношении Российской Федерации, регистрация нового аккаунта в AWS и получение ключей доступа (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY) оказались невозможны. Попытки регистрации с использованием VPN не увенчались успехом – письма для подтверждения не приходили, а система безопасности AWS блокировала доступ.

В связи с этим практическое развёртывание ресурсов в AWS не производилось. Теоретически, при наличии действующего аккаунта и корректно настроенных переменных окружения, конфигурация Terraform позволила бы создать необходимую инфраструктуру.

Код, представленный в папке DZ_1/aws/, полностью соответствует требованиям задания (VPC, публичная и приватная подсети, Internet Gateway, NAT Gateway, группы безопасности, ВМ с публичным и приватным IP).
